# IQM2GLB PROJECT STATUS

## 1. Project Overview
A bidirectional model converter for Quake-family games (Warsow/Sovereign).
Originally designed for IQM -> GLB, now supports multiple formats and deep hierarchy preservation.

## 2. Supported Formats
- **IQM**: Inter-Quake Model (Z-up, "Backward" CW winding).
- **GLB/GLTF**: GL Transmission Format (Y-up, Standard CCW winding).
- **SKM/SKP**: Warsow legacy skeletal format (Z-up, "Backward" CW winding).
- **Binary FBX**: Autodesk FBX (Y-up, Standard CCW winding). Native high-performance implementation.
- **Universal**: Assimp-based loading for supplementary formats.

## 3. Internal Representation (Intermediate Model)
To maintain consistency across converters, all loaders/writers must adhere to:
- **Coordinate System**: Y-up (Right-Handed).
- **Winding Order**: Standard CCW (Counter-Clockwise).
- **Animation**: Sample-based dense frames at 30 FPS (default).

## 4. Loader/Writer Capabilities
### Loaders:
- **cgltf**: Fast, lean GLB/GLTF loading.
- **ufbx (Primary)**: Standard-grade FBX loading with full hierarchy and animation support.
- **Assimp**: Robust, handles complex hierarchy and sampling normalization (non-FBX).
- **SKP**: Specialized loader for legacy SKM/SKP pairs.
- **IQM**: Native IQM loader with Z-up to Y-up conversion and winding correction.

### Writers:
- **Binary FBX (Primary)**: Native binary writer with modern "v7-Final" standards: single-armature hierarchy, full name sanitization, and relative timing for perfect Blender compatibility.
- **cgltf (Standard)**: Original fast GLB writer, normalized for CCW.
- **IQM**: Native writer with Y-up to Z-up conversion and winding reversal (CCW -> CW).

## 5. Critical Implementation Details
- **Frame Drift Fix**: Use `std::round(max_t * fps) + 1` for frame count calculation to avoid floating-point inflation.
- **GLTF Duration**: Animations must have duration > 0. For 1-frame animations, set duration to 0.001.
- **Hierarchy Restoration**: GLB loader restores the full node tree (not just joints) to preserve animation roots.
- **IBM Handling**: Inverse Bind Matrices (IBMs) are loaded directly from GLB to ensure skinning stability.
- **Winding**: Internal state is strictly **Y-up CCW**. Formats like IQM are flipped during Load/Save to match.

## 6. FBX Implementation Notes (Modern Standards)
- **Joint Spinning Fixed**: Implemented "Continuous Euler Extraction" in `fbx_writer.cpp`. By picking the nearest solution and seeding from the bind pose, animation curves remain smooth through gimbal lock.
- **Root Hierarchy**: A single clean Armature object is created in Blender by ensuring all root joints and meshes are siblings under the scene root (parent 0).
- **BindPose Integration**: All joints and the mesh are included in the `BindPose` (`PoseNode`) to ensure Blender correctly integrates bones into the `Pose` section of the object.
- **Name Sanitization**: Spaces are replaced with underscores (`_`) in both node names and animation channels to ensure exact binding in Blender.
- **Mesh/Bone Distinction**: Mesh nodes are suffixed with `_mesh` to prevent name collisions with bones of the same name.
- **Scale Control**: `UnitScaleFactor` is set to 100.0 to match Blender's meter/centimeter expectations.
- **Relative Timing**: Animation channels start at time 0.0s for every take, preventing offset issues in modern importers.
- **Animation Deduplication**: Names in `animation.cfg` are used to deduplicate/override internal animations (no more `.001` intruders).

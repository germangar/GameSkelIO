# IQM2GLB Multi-Fix Report - March 13, 2026

Summary of improvements and bug fixes implemented in the iqm2glb subproject:

## 1. Corrected Weighted Vertex/Normal Reconstruction (skp_loader.cpp)
- **Issue**: The `transform_vec` and `rotate_vec` functions were incorrectly weighting only parts of the transformation (e.g., only translation) or not taking weight at all.
- **Fix**: Re-implemented these functions to properly apply the influence weight to all components of the resulting vector after the matrix operation. 
- **Impact**: Fixes distorted meshes when loading SKM/SKP models with multiple bone influences.

## 2. Fixed BindPose Node Counts (fbx_writer.cpp)
- **Issue**: An off-by-one error in `num_pose_nodes` was causing FBX files to declare more PoseNodes than were actually being written.
- **Fix**: Corrected the count calculation to precisely match the number of joints plus the mesh model.
- **Impact**: Improved compatibility with strictly conforming FBX importers.

## 3. Memory Leak and Safety Fixes (glb_writer.cpp)
- **Issue**: The GLB writer was using `calloc` for many structures without ever calling `cgltf_free`. Additionally, string literals were assigned to pointers that the deallocator would attempt to free, leading to potential crashes.
- **Fix**: 
    - Added a `cgltf_free(out)` call before returning from `write_glb`.
    - Ensured all string assignments use `strdup` or dynamic allocation.
- **Impact**: Significant reduction in memory usage during batch conversions and increased stability.

## 4. Enhanced Skeleton Architecture (model.h & main.cpp)
- **Issue**: Skeleton data from formats like GLB/FBX can have joints in arbitrary order, but IQM and internal bind-pose calculations require parents to appear before children.
- **Fix**: 
    - Upgraded `validate_skeleton()` to verify topological sorting.
    - Implemented `reorder_skeleton()` to automatically sort the hierarchy and re-map all vertex joint indices and animation tracks.
- **Impact**: Robust support for out-of-order skeletons from modern DCC tools.

## 5. General Code Cleanup
- **Fix**: Removed unused variables (`qfusion` in main.cpp, `skp_fps` in skp_loader.cpp) to eliminate compiler warnings.
- **Verification**: Verified a clean build of iqm2glb.exe using the existing Makefile and MinGW64.

## 6. Unique Output Filenaming (main.cpp)
- **Feature**: Added a `get_unique_path` helper function.
- **Functionality**: Before writing the output file, the tool now checks if a file with the target name already exists. If so, it appends a numeric suffix (e.g., `_1`, `_2`) to prevent overwriting existing work.
- **Impact**: Safely allows multiple conversion attempts without losing previous results.

## 7. Animation Stability: Quad-Branch Euler Solver (math_utils.h)
- **Issue**: Standard Euler extraction collapses at gimbal lock ($Y = \pm 90^\circ$), causing large rotational jumps (72+ degrees) in the Pelvis and root joints during conversion.
- **Fix**: Implemented a "Quad-Branch Solver" that evaluates four separate Euler candidates per frame, including gimbal-stabilized solutions with pinned redundancy. It selects the candidate that minimizes coordinate distance from the previous frame.
- **Impact**: Provides perfectly smooth animation curves through all orientations, eliminating flickering in the exported FBX/GLB files. Verified with 135-channel audit.
